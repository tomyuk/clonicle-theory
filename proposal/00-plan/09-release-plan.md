# Release Plan

## 目的

本成果物は完成版を一度だけ公開するのではなく、Proposalとして段階的に公開し、批評と実践を取り込みながら成熟させます。各版では、内容の完成度だけでなく、理論上の確度、検証状況、公開対象、期待するフィードバックを明示します。

## Version 0.1　Internal Proposal

企業存在論、Chronicle、Brainful、Protocol、Allianceの基礎構造を作成し、用語と論理順序を確認します。主な対象は執筆者と近接協力者です。事実確認よりも、概念の整合性と元議論からの逸脱検出を優先します。

## Version 0.3　Partner Review

士業、金融機関、経営者、AI・IT専門家などの限定的なレビューを受けます。現場語彙との接続、誤解されやすい表現、役割分担、実装可能性を確認します。この段階でも提案概念を実証済みとして扱いません。

## Version 0.5　White Paper Draft

Part単位の本文、主要図表、参考文献、用語集を統合します。公開可能なホワイトペーパー草稿として、理論的背景、適用範囲、限界、未解決事項を明示します。

## Version 0.8　Public Review

公開レビュー版として配布し、批評、反例、事例、制度上の課題、実装上の懸念を収集します。Issue、Discussion、対話会、セミナーなど複数の経路を用い、変更履歴とRDEレビューを残します。

## Version 1.0　Proposal Publication

社会的提案として一定の整合性を持つ版です。完成した教義ではなく、実践と検証へ移行できる基準版として位置付けます。Markdown、PDF、説明用スライド、概要版を同期させます。

## Version 1.x　Practice Updates

コンサルティング、地域金融、人材育成、事業承継、AI番頭などの実践から得られた知見を反映します。理論とケーススタディを区別し、成功例だけでなく失敗・非適合事例も記録します。

## Version 2.0　Book / Protocol Specification

公開レビューと実践結果を踏まえ、書籍版または体系的なProtocol仕様へ発展させます。White Paperの文章を単純に増補するのではなく、読者、目的、証拠水準に応じて再構成します。

## 公開形式

- GitHub上のMarkdown原稿
- 役員・金融機関向けPDF
- パートナー説明用スライド
- 一般向け概要版
- 図表・用語集
- 参考文献データ
- 変更履歴とRDEレビュー

## リリース判定

各版の公開前に、用語の一貫性、論理構造、外部事実の確認、RDE差異検証、図表との整合、アクセシビリティ、ライセンスと著作権表示を確認します。
